회원가입 및 로그인

회원가입 시 필요한 정보

1. ID

- key data로 활용

2. 비밀번호

- 암호화 해서 저장

3. 비밀번호 확인

- 비밀번호와 비밀번호 확인이 일치하는지 확인


로그인 시 필요한 정보

1. ID

- key data로 활용

2. 비밀번호

- 입력받은 비밀번호를 암호화해서 데이터베이스와 대조


구현 방법

1. 세션

- 모든 페이지에서 세션을 시작하도록 만들고 로그인 성공 시 세션에 ID(username)를 저장
로그인 된 상태면 네비게이션바에 로그인, 회원가입 버튼 대신 아이디와 로그아웃 버튼이 보이도록 함

2. 비밀번호

- 비밀번호는 평문으로 그대로 저장시 심각한 보안문제가 발생할 수 있기 때문에 암호화해서 저장
- 로그인 시에도 사용자가 입력한 비밀번호를 암호화해서 데이터베이스에 저장된 암호화된 비밀번호와 대조


구현 중 생긴 문제

1. 회원가입 데이터 오류

- 회원 가입 시 db에 데이터가 제대로 전달되지 않는 오류가 생겨서 해결에 어려움을 겪음
- 테이블의 타입설정이 잘못되어서 발생한 문제였음

2. 비밀번호

- 비밀번호를 암호화해서 저장은 했는데 로그인 시에 대조할 방법을 잘 몰랐음
- 입력받은 비밀번호를 암호화하면 가입시에 암호화해서 데이터베이스에 저장된 비밀번호와 대조할 수 있음
